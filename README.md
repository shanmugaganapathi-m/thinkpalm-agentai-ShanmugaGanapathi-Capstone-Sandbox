# Personal Finance Analyzer

An agentic AI system that reads Indian bank statement PDFs, extracts every transaction, and produces intelligent spending insights, anomaly detection, and personalised budget recommendations powered by Claude.

---

## Features

| Feature | Details |
|---|---|
| **PDF Parsing** | Extracts transactions from any Indian bank PDF (IndusInd, HDFC, ICICI, Axis, SBI, …) |
| **Multi-format dates** | Handles `DD-Mon-YYYY`, `DD/MM/YYYY`, `DD-MM-YYYY`, `YYYY-MM-DD`, and more |
| **12 spending categories** | Auto-classifies Food, Groceries, Transport, EMI, Insurance, Utilities, and more |
| **Claude AI recommendations** | 4–5 personalised tips generated by `claude-sonnet-4-6` with exact ₹ amounts |
| **Anomaly detection** | Flags large transactions (>₹10k), category spikes (>150% of 3-month avg), duplicates |
| **Persistent memory** | Stores up to 12 months of history; deduplicates by month automatically |
| **Month-over-month comparison** | Trend charts and category delta tables across sessions |
| **Graceful fallback** | Rule-based recommendations when no API key is supplied |

## Setup Guide

### 1. Prerequisites

- Python **3.11** or higher
- If the AI analysis needed, An **Anthropic API key** — get one at [console.anthropic.com](https://console.anthropic.com)
- You can still run the analysis tool, that will give analysis based on rules.

### 2. Clone the repository

```bash
git clone https://github.com/<your-username>/personal-finance-analyzer.git
cd personal-finance-analyzer
```

### 3. Create a virtual environment

```bash
# Create
python -m venv venv

# Activate — macOS / Linux
source venv/bin/activate

# Activate — Windows (Command Prompt)
venv\Scripts\activate.bat

# Activate — Windows (PowerShell)
venv\Scripts\Activate.ps1
```

### 4. Install dependencies

```bash
pip install -r requirements.txt
```

### 5. Set your API key (optional but recommended)

Setting the key in the environment means you won't have to paste it in the UI on every run.

**macOS / Linux**
```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

**Windows (Command Prompt)**
```cmd
set ANTHROPIC_API_KEY=sk-ant-...
```

**Windows (PowerShell)**
```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-..."
```

**Or use a `.env` file** (loaded automatically via `python-dotenv`):
```bash
cp .env.example .env
# Open .env and set ANTHROPIC_API_KEY=sk-ant-...
```

> **Note:** You can also paste the key directly in the app's **Analysis Settings** panel on the Upload tab — no environment variable needed.

### 6. Run the application

```bash
streamlit run app.py
```

The dashboard opens at **http://localhost:8501**.

## Using the App

### Step 1 — Upload & Parse

1. Go to the **Upload & Parse** tab.
2. *(Optional)* Expand **Analysis Settings** on the right to choose your AI provider, model, and paste your API key.
3. Drop your PDF bank statement into the upload area.
4. Click **Parse Statement**.

The parser extracts all transactions and immediately generates insights.

### Step 2 — Dashboard

Switch to the **Dashboard** tab to see:
- Income, spend, net savings, and savings rate for the month
- Donut chart of spending by category
- Horizontal bar chart of category breakdown
- Anomaly alert and top 3 recommendations

### Step 3 — Recommendations

The **Recommendations** tab shows:
- A **✦ Powered by Claude AI** badge when Claude generated the tips (requires API key)
- A **⚙ Rule-based analysis** badge when the fallback ran (no key or API error)
- 4–5 personalised tips referencing your actual ₹ amounts
- Quick-win actions and a savings opportunity callout

### Step 4 — History

Each parsed statement is saved to memory automatically. The **History** tab shows month-over-month comparisons, spending trend charts, and category change tables once you have 2+ months of data.

## Supported Bank Statement Formats

The parser handles multi-line PDF layouts (where each transaction spans 4–6 lines) and all common Indian bank date formats:

| Format | Example |
|---|---|
| `DD-Mon-YYYY` | `02-Mar-2026` |
| `DD/MM/YYYY` | `02/03/2026` |
| `DD-MM-YYYY` | `02-03-2026` |
| `DD.MM.YYYY` | `02.03.2026` |
| `DD Mon YYYY` | `02 Mar 2026` |
| `YYYY-MM-DD` | `2026-03-02` |
| 2-digit year variants | `02/03/26` |

**Tested with:** IndusInd Bank. Should work with HDFC, ICICI, Axis, SBI, and any PDF where transactions follow a date → description → amount layout.

## Architecture

```
PDF file
   │
   ▼
ParserAgent.run()
   ├── validate_pdf()          — checks file is a readable PDF
   ├── read_pdf()              — extracts raw text via PyMuPDF
   ├── normalize_transactions() — multi-line block grouping + date normalisation
   └── save_memory()           — upserts session summary (deduplicates by month/year)
   │
   ▼
InsightsAgent.run()
   ├── calculate_monthly_summary()  — income, spend, savings rate, per-category totals
   ├── compare_with_history()       — category deltas vs 3-month average
   ├── detect_anomalies()           — spikes, large transactions, duplicates
   └── generate_recommendations()  — Claude API call (falls back to rule-based)
   │
   ▼
Streamlit Dashboard
   ├── Upload & Parse tab   — file upload + inline settings
   ├── Dashboard tab        — KPI cards + charts + top recommendations
   ├── History tab          — trend chart + MoM comparison
   ├── Anomalies tab        — flagged transaction cards
   ├── Recommendations tab  — Claude-powered tips + quick wins
   └── Memory tab           — stored sessions + CSV export
```
